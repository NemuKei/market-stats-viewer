# Purpose
新規ファイルの誤コミット（生成物・ローカル設定・機密断片）を防ぎ、再現性のある管理対象だけを Git 管理下に残す。
root `AGENTS.md` の repo-wide ルールを前提にし、この Skill は `.gitignore` 判定に必要な task-specific procedure だけを追加する。

# When to use / When NOT to use
- When to use:
  - 新しくファイル/ディレクトリを作成した直後
  - ビルド・実行・更新スクリプトで新しい生成物が出たとき
- When NOT to use:
  - 既存追跡ファイルの内容更新のみで、新規ファイルがないとき
  - 仕様として永続管理するファイル（正本ドキュメント・実装コード・入力データ）を追加したとき

# Procedure
1. 新規ファイルを列挙する（`git status --short` を使用）。
2. 各ファイルを次の優先順で判定する。
   - **除外（.gitignore へ）**: 一時ファイル、キャッシュ、ビルド生成物、ローカル環境依存ファイル、資格情報を含む可能性があるもの
   - **追跡（.gitignore へ入れない）**: ソースコード、仕様書、再現に必要な設定、共有すべき入力データ
3. 除外対象のみ `.gitignore` へ最小差分で追記する（既存ルールの重複追加はしない）。
4. 判定理由を 1 行で残す（PR 説明または作業ログ）。

# Decision Heuristics
- 原則: 「他者環境で再生成できるもの」は除外、「リポジトリの正本として必要なもの」は追跡。
- 迷った場合:
  - セキュリティ懸念（鍵・token・cookie・個人情報）が 1 つでもあれば除外
  - 再現性に必要で代替取得がないなら追跡
  - 判断不能なら追加せず、用途を確認してから決定

# Validation
- `git status --short` で、意図しない生成物が追跡候補に残っていないこと。
- `.gitignore` に重複パターンがないこと。
- 除外理由を説明できること（セキュリティ / 再現性 / 生成物のいずれか）。
