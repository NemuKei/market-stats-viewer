# Purpose
ドキュメント運用を重複なく維持し、正本反映をファイル更新で確定する。

# When to use / When NOT to use
- When to use: 新規ドキュメント作成判断 / 重複整理 / 壁打ち内容の正本化
- When NOT to use: 草案共有のみで正本反映未決定 / 3条件を満たせない

# Procedure
1. 変更内容を既存ドキュメントへ責務分離できるか確認する。
2. 新規作成が必要な場合は3条件を**すべて**確認:
   1. 既存ドキュメントに責務分離できない
   2. 今後2回以上参照する見込み
   3. 所有者と更新トリガーを定義可能
3. 不足 → 新規作成しない。既存へ統合。
4. 同一ルールが複数箇所 → 正本を1箇所に定め、他は短いリダイレクト文へ置換。
5. 壁打ち会話は非正本。正本化するなら必ずファイル更新で確定。

# Validation
- 3条件充足 / 所有者+トリガー定義済 / 重複なし / 正本はファイル更新で確定
