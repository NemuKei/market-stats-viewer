# Purpose
常設コンテキストへの反映可否を、条件に基づいて判断し、必要時のみ最小差分で反映する。

# When to use / When NOT to use
- When to use:
  - 常設コンテキストに残すかどうかを判断するとき
  - 将来の意思決定に影響する情報を正本へ反映するとき
- When NOT to use:
  - 単発メモや一時的な会話内容だけを扱うとき
  - 4条件を満たせないとき
  - 保存先を1ファイルに特定できないとき

# Inputs
- 反映候補の情報（ユーザーが明示した内容）
- 保存先候補の正本ファイル（`STATUS.md` / `DECISIONS.md` 相当など）
- 矛盾判定に必要な同順位の情報源

# Outputs
- 更新済みの正本ファイル（1ファイル）
- 未解決矛盾がある場合のみ `DECISIONS.md` 相当への `D-YYYYMM-xxx` 暫定記録

# Procedure
1. 反映候補が次の4条件をすべて満たすか確認する。
   1. 今後2回以上の再利用が見込める。
   2. 将来の意思決定に影響する。
   3. 1〜3行で要点化できる。
   4. 保存先を1ファイルに特定できる。
2. 1つでも満たさない場合は反映しない。
3. 反映する場合は、明示された内容だけを最小差分で更新する（推測で補完しない）。
4. 同順位の情報源で矛盾があり未解決なら、`DECISIONS.md` 相当に `D-YYYYMM-xxx` 形式で暫定記録し、より新しい決定を優先して進める。

# Validation
- 4条件を満たす根拠がある。
- 更新先が1ファイルに特定されている。
- 推測による追記がない。
- 未解決矛盾時は `D-YYYYMM-xxx` で記録されている。

# Examples
- 「この判断を常設コンテキストに残すか、4条件で判定して」
- 「この内容を推測なしで `DECISIONS.md` に最小差分で反映して」
